# NotebookLMへのアップロードガイド

このガイドでは、Vib SNSのドキュメントをNotebookLMにアップロードして、AIによる質問応答システムを構築する方法を説明します。

---

## 用意したドキュメント

1. **vib_sns_complete_documentation.md**
   - アプリの完全なドキュメント
   - 概要、技術スタック、データモデル、サービス、状態管理、Cloud Functions、Firestoreルール、画面構成、認証フロー、FAQ、用語集

2. **api_reference.md**
   - API技術リファレンス
   - Firebase、位置情報、BLE、地図、画像処理などのAPI詳細とコードサンプル

---

## NotebookLMへのアップロード手順

### ステップ1: NotebookLMにアクセス
1. [NotebookLM](https://notebooklm.google.com/) にアクセス
2. Googleアカウントでログイン

### ステップ2: 新しいノートブックを作成
1. 「新しいノートブック」をクリック
2. ノートブック名を「Vib SNS ナレッジベース」などに設定

### ステップ3: ソースをアップロード
1. 「ソースを追加」または「+」ボタンをクリック
2. 以下のファイルをアップロード:
   - `docs/notebooklm/vib_sns_complete_documentation.md`
   - `docs/notebooklm/api_reference.md`
3. 必要に応じて `README.md` も追加可能

### ステップ4: 共有設定
1. 画面右上の「共有」ボタンをクリック
2. 共有したい人のメールアドレスを入力
3. 権限を選択（「閲覧者」または「編集者」）
4. 「リンクをコピー」で共有リンクを取得

## 質問の例

NotebookLMにアップロード後、以下のような質問ができます：

### アプリ全般
- 「Vib SNSはどんなアプリですか？」
- 「このアプリの主な機能は何ですか？」
- 「対応しているプラットフォームは？」

### 技術的な質問
- 「使用している技術スタックを教えてください」
- 「状態管理には何を使っていますか？」
- 「認証方式は何がありますか？」

### データモデル
- 「Profileモデルのフィールドを教えてください」
- 「タイムライン投稿はどのようなデータ構造ですか？」
- 「Firestoreのコレクション構造を説明してください」

### 機能詳細
- 「すれちがい検出の仕組みを説明してください」
- 「ハッシュタグマッチングとは何ですか？」
- 「エモーションマップ機能について教えてください」

### 開発関連
- 「プロジェクトのセットアップ手順を教えてください」
- 「必要な権限は何ですか？」
- 「Cloud Functionsは何をしていますか？」

### API関連
- 「位置情報の取得方法を教えてください」
- 「BLEスキャンはどのように実装されていますか？」
- 「Firestoreへのデータ保存方法を教えてください」

## 追加ドキュメントの提案

より詳細な質問に対応するため、以下のドキュメントを追加することを検討できます：

1. **画面ごとのUI仕様書**
   - 各画面のワイヤーフレーム
   - UI/UX設計の詳細

2. **開発履歴・変更ログ**
   - 各バージョンの変更内容
   - バグ修正履歴

3. **運用マニュアル**
   - デプロイ手順
   - 監視・アラート設定

4. **トラブルシューティングガイド**
   - よくある問題と解決方法

## 注意事項

- NotebookLMは現在、テキストベースのドキュメント（PDF、Google Docs、テキストファイル）に対応しています
- Markdownファイルは直接アップロードできる場合と、テキストとしてコピー&ペーストが必要な場合があります
- 機密情報が含まれる場合は、共有設定に注意してください
- ドキュメントを更新した場合は、ソースを再アップロードする必要があります
